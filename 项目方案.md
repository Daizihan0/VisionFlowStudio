# Windows 视觉定位流程自动化程序方案

## 1. 目标结论

本项目可行，建议首版定位为：

- 面向 `Windows 10/11 x64` 的桌面端可视化流程自动化工具
- 支持截图定位、像素定位、手动十字定位
- 支持鼠标、键盘、快捷键、滚轮等硬件级输入模拟
- 支持节点连线式流程图编辑
- 支持分支、重试、跳转、循环、定时执行
- 支持录制一段用户操作，并将其作为流程节点或子流程复用
- 首版不做 OCR，但架构预留后续接入能力
- 以“普通办公软件自动化”为主要目标场景

建议技术路线：`C# + .NET 8 + WPF + Win32 API + OpenCvSharp`。

原因：

- 原生适配 Windows 桌面体验，UI 表现好
- 便于调用 `SendInput`、窗口枚举、屏幕捕获、全局热键等底层能力
- 支持打包为独立运行版本，不依赖用户手动安装运行环境
- 后续扩展 OCR、插件、脚本节点、任务计划等功能比较顺滑

## 2. 最终体积预估

按本方案完成后，若发布为 `Windows x64 独立运行版`，体积大致如下：

### 推荐发布形式：便携版目录

- `Release` 目录整体：约 `100 MB ~ 180 MB`
- 主程序 `exe`：通常 `5 MB ~ 20 MB`
- 其余为 `.NET Runtime`、WPF 依赖、OpenCV 原生库、资源文件等

### 若强行打成单文件 EXE

- 单文件 `exe`：约 `120 MB ~ 200 MB`
- 首次启动可能会有解包或预热，稳定性和排障不如便携目录版直观

### 为什么会比较大

主要不是业务代码大，而是这些部分占体积：

- `.NET 8 Windows Desktop Runtime`
- `WPF` 运行库
- `OpenCvSharp` / `OpenCV` 原生 DLL
- 图标、截图、录制、日志、流程模板等资源

### 我建议的交付方式

建议同时提供两种：

- `便携目录版`：最稳，双击即可运行，便于排障
- `单文件版`：方便分发，但作为附加版本

如果后续不使用 OpenCV，而改成纯手写图像匹配，体积能再小一些，但性能、稳定性和开发效率都会下降，不建议。

## 3. 首版核心功能范围

### 3.1 定位能力

1. **截图定位**
   - 用户点击“截图定位”
   - 框选屏幕区域
   - 自动保存为模板图
   - 执行时在全屏或指定区域查找该图像
   - 支持相似度阈值、灰度匹配、重试次数、查找超时

2. **剪贴板贴图定位**
   - 用户可直接把截图粘贴到程序内
   - 程序将图片作为模板图使用
   - 支持命名、预览、替换

3. **手动十字定位**
   - 用户点击“手动定位”
   - 屏幕上显示十字准星与放大镜预览
   - 用户拖到目标点确认
   - 保存绝对坐标或相对窗口坐标

4. **像素定位**
   - 记录某点颜色值
   - 支持容差判断
   - 用于判断某一步是否完成或某状态是否出现

5. **窗口锚定**
   - 可选绑定目标窗口
   - 坐标以窗口客户区为参考
   - 窗口移动后流程仍可继续工作

### 3.2 动作能力

1. **鼠标动作**
   - 移动到目标
   - 左键单击
   - 左键双击
   - 右键点击
   - 中键点击
   - 滚轮上滚/下滚
   - 按下/抬起/拖拽

2. **键盘动作**
   - 硬件级按键模拟（基于 `SendInput`）
   - 输入文本
   - 单键、组合键、连发键
   - 快捷键：如 `Ctrl+C`、`Ctrl+V`、`Alt+Tab` 等
   - 支持按下时长、键间隔、重复次数

3. **等待动作**
   - 固定等待若干毫秒/秒
   - 等待图片出现
   - 等待图片消失
   - 等待像素变为目标颜色
   - 等待窗口出现/激活

### 3.3 流程控制能力

1. **顺序执行**
2. **条件分支**
   - 如果图片出现则走 A
   - 否则走 B
3. **重试机制**
   - 失败后重试 N 次
   - 每次重试前等待 X ms
4. **跳转机制**
   - 跳转到指定节点
   - 跳转到标签
5. **循环机制**
   - 固定次数循环
   - 条件满足前持续循环
6. **异常处理**
   - 节点失败后停止
   - 失败后跳转备用分支
   - 失败后截图并记录日志
7. **子流程/录制片段复用**
   - 录制一段操作保存为可复用节点
   - 在多个流程中重复调用

### 3.4 可视化能力

1. **节点连线流程图编辑器**
   - 中央画布显示流程图
   - 节点拖拽排布
   - 节点之间连线
   - 条件分支显示不同出口
   - 当前运行节点高亮

2. **属性面板**
   - 选中节点后编辑参数
   - 坐标、模板图、阈值、超时、分支目标等均可配置

3. **资源面板**
   - 查看已保存的截图模板、像素点、录制片段

4. **运行面板**
   - 显示当前流程状态
   - 显示当前节点、运行日志、失败原因、重试次数

### 3.5 调试与安全能力

1. **全局紧急停止热键**
   - 例如 `F10` 立即停止全部执行
2. **单步调试**
   - 一步一步执行节点
3. **断点**
   - 在某节点暂停
4. **运行日志**
   - 每个节点记录开始、结束、耗时、结果
5. **失败截图**
   - 出错时自动保存现场截图
6. **DPI / 多屏适配**
   - 处理高分屏、主副屏坐标转换
7. **管理员权限提醒**
   - 当目标程序为管理员权限时提示用户以管理员身份运行本程序

### 3.6 定时执行

1. 指定时间执行
2. 每天/每周定时执行
3. 启动时最小化到托盘
4. 到点自动运行指定流程
5. 运行结束后通知

### 3.7 录制模式

录制模式建议做成“半自动录制”，而不是无脑录全部底层事件。

录制内容：

- 鼠标点击
- 鼠标移动关键点
- 双击、右键、滚轮
- 键盘按键
- 等待时间
- 可手动在录制后补充“视觉判断节点”

建议录制后的处理：

- 自动生成节点序列
- 允许用户将纯坐标节点替换为视觉定位节点
- 可把录制结果封装为“子流程节点”

这样更稳，不容易因为坐标漂移导致流程失效。

## 5. 技术架构方案

## 5.1 技术选型

- UI：`WPF`
- 运行时：`.NET 8`
- 架构模式：`MVVM`
- 图像匹配：`OpenCvSharp`
- 底层输入：`Win32 SendInput`
- 截图/屏幕采集：`GDI / Windows Graphics Capture`（首版可先用 GDI）
- 定时调度：应用内调度器 + Windows 启动项/托盘支持
- 数据存储：`JSON`
- 日志：文本日志 + 运行记录 JSON

## 5.2 模块拆分

1. **UI Shell**
   - 主窗口
   - 侧边栏
   - 流程图画布
   - 属性编辑面板
   - 资源库面板
   - 日志面板

2. **Flow Designer**
   - 节点模型
   - 连线模型
   - 画布交互
   - 节点拖拽
   - 撤销/重做

3. **Execution Engine**
   - 节点调度
   - 条件判断
   - 重试/超时
   - 跳转/循环
   - 子流程调用

4. **Vision Engine**
   - 截图模板管理
   - 图像查找
   - 像素判断
   - 坐标换算
   - 窗口锚定

5. **Input Engine**
   - 鼠标模拟
   - 键盘模拟
   - 快捷键组合
   - 拖拽与滚轮

6. **Recorder Engine**
   - 钩子采集用户输入
   - 事件归并
   - 自动生成节点

7. **Scheduler**
   - 定时规则管理
   - 托盘启动
   - 到点运行流程

8. **Project Storage**
   - 流程文件
   - 模板图资源
   - 录制片段
   - 日志
   - 用户设置

## 8. 关键难点与解决策略

### 8.1 高 DPI 与多屏坐标

问题：不同屏幕缩放会导致坐标偏移。

方案：

- 程序声明 `PerMonitorV2 DPI Aware`
- 所有定位统一保存物理像素与逻辑坐标映射
- 在执行前进行屏幕坐标换算

### 8.2 视觉匹配稳定性

问题：界面轻微变化时纯像素匹配容易失败。

方案：

- 支持阈值调节
- 支持灰度匹配
- 支持缩小搜索区域
- 支持“中心点击 + 偏移点击”
- 提供匹配结果预览

### 8.3 录制过于脆弱

问题：录到大量绝对坐标后复现稳定性差。

方案：

- 录制后自动清洗事件
- 连续移动归并为关键动作
- 允许将某些点击转换为“图像定位点击”
- 推荐用户对关键节点补视觉锚点

### 8.4 权限问题

问题：目标程序若以管理员身份运行，普通权限程序无法稳定注入输入。

方案：

- 启动时检测权限级别
- 必要时提示重启为管理员权限

## 11. 首版验收标准

当以下能力都能稳定工作时，可认为首版达标：

- 能创建一个节点连线流程图
- 能保存并重新打开项目
- 能通过截图找到某按钮并点击
- 能通过手动定位坐标点击某位置
- 能输入文本和组合快捷键
- 能等待图片出现后再执行下一步
- 能在失败后自动重试或跳转备用分支
- 能录制一小段操作并回放
- 能定时启动某个流程
- 能用热键立即停止执行
